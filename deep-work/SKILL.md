# Deep Work

## 레이어 위치

**L2 (환경·시간)** — pomodoro-and-focus와 같은 층. L3~L6의 모든 학습이 일어나는 *컨테이너*. L2가 무너져 있으면 상위 레이어 무의미. 초심자·집중 붕괴 상태에선 pomodoro로 먼저 훈련 후 deep work.

## 한 줄 요약

**집중이 필요한 인지 작업(deep work)과 단순 처리(shallow work)를 구분하고, 깊은 일을 위한 *보호된 시간 블록*을 구조화하라.** 주의는 근육이며, 훈련해야 발달한다.

## 이론·배경

- **Cal Newport** — 조지타운 컴퓨터공학 교수. 학술 연구 + 대중 저술. *Deep Work* (2016), *A World Without Email* (2021), *Slow Productivity* (2024).
- Newport의 독자적 이론은 아니며, 다음을 종합:
  - **Flow** (Csikszentmihalyi 1990) — 몰입 상태 연구.
  - **Deliberate Practice** (Ericsson) — 고도 집중 연습.
  - **Attention residue** (Sophie Leroy 2009) — 작업 전환 시 잔여 주의.
  - **Erik Brynjolfsson 등의 지식 경제 분석** — 왜 집중력이 경제적 자산인가.
- 실증: Newport는 이론가·에세이스트에 가깝고, 본인 책도 사례·내러티브 중심. 기저 연구는 견고하지만 "Deep Work" 자체는 통합 프레임워크.

## 핵심 구분

### Deep Work
- 인지적으로 요구되는 작업
- 방해 없는 조건에서 한계 근처 수행
- 가치 창출·새 스킬 습득
- 쉽게 복제 불가
- 예: 복잡한 코드 설계·논문·분석·글쓰기·학습

### Shallow Work
- 인지적 부담 낮음
- 방해 속에서도 수행 가능
- 가치 낮고 쉽게 복제 가능
- 예: 이메일 응답·단순 회의·상태 업데이트·메시지 확인

**문제**: 지식 노동자 대부분이 shallow에 압도되어 deep 시간이 거의 0. Newport는 "오늘 업무의 몇 %가 deep인가?"를 추적해보라고 제안.

## 네 가지 규율 (Philosophies)

Newport는 deep work를 일상에 통합하는 네 양식을 구분:

### 1. Monastic (수도원식)
- 외부 접촉을 거의 차단, 긴 기간 deep work에만 집중.
- 예: Donald Knuth — 1990년부터 이메일 안 함. *Art of Computer Programming* 집필에만 전념.
- 적용 가능: 소수의 독립 연구자·작가. 대다수엔 비현실적.

### 2. Bimodal (양분식)
- 한 주·한 달의 일정 블록은 완전 집중, 나머지는 shallow 허용.
- 예: Carl Jung이 집필용 돌탑에 며칠~몇 주 격리, 나머지 시간은 환자·가족.
- 적용: 프로젝트성 작업자·교수·상담가.

### 3. Rhythmic (리듬식)
- 매일 정해진 시간대(예: 오전 6~9시) deep work.
- "Jerry Seinfeld 체인" — 달력에 X 표시, 끊지 않기.
- 가장 흔하고 현실적. 일상 직장인에 맞음.

### 4. Journalistic (저널리즘식)
- 일정이 예측 불가능할 때 *기회가 열리면 즉시* 깊게 들어감.
- Walter Isaacson이 집필을 출장 중 30분 짬에서도 이어갔다는 사례.
- 고난도. 주의 스위칭 능력이 이미 훈련된 사람만.

대다수에게 **Rhythmic**이 출발점. Bimodal은 프로젝트 집중기에. Monastic·Journalistic은 예외 사례.

## 관련 개념: Attention Residue

- **Sophie Leroy (2009)** "Why is it so hard to do my work?" *Organizational Behavior and Human Decision Processes.*
- 작업 A에서 B로 전환 시, A의 인지 잔여물이 B 수행을 방해.
- 특히 A가 *미완*으로 끊길 때 잔여 큼.
- 함의: 10분마다 메신저 확인은 실제 deep work 시간을 분해.
- **(→ `pomodoro-and-focus` 참조)**

## 실전 규칙 (Newport의 제안)

### 1. 일정 블록
- 하루를 블록 단위로 사전 계획. 각 블록 = 1~4시간.
- deep work 블록은 *회의·알림 없음*.

### 2. 수량화
- 주간 deep work 시간 기록. 목표 설정(예: 주 15시간).
- 시간이 아니라 *깊이의 총량* 추적.

### 3. 커뮤니케이션 규칙
- 이메일 확인을 하루 2~3회로 제한.
- 실시간 메신저 최소화 — "상시 온라인"은 shallow를 기본값으로 만듦.
- 예측 가능한 응답 시간 설정.

### 4. 지루함 허용
- 주의 파편화의 원인 중 하나는 *지루함 내성 저하*. 줄서기·엘리베이터 시간에 폰 꺼내는 습관이 deep work 능력도 갉아먹음.
- 일부러 "폰 없이 대기" 연습.

### 5. 소셜미디어 축소
- 자산 창출에 필요한 최소만 사용. Newport는 본인 SNS 전혀 안 함.
- 현실 맥락 차이 존재. 완전 배제가 아니라 *의식적 사용*.

### 6. Shallow 시간 상한
- Shallow work도 필요하지만 *상한*을 두어 deep을 보호.

## 한국 맥락 예시

**직장인 개발자 (Rhythmic)**:
- 오전 9:30~11:30 = deep block. Slack 꺼두고 한 기능 구현에만 집중.
- 11:30~12:00 = 메신저·이메일 일괄 응답.
- 오후 1:30~3:00 = 또 다른 deep block (코드 리뷰·설계).
- 오후 나머지 = 회의·PR 리뷰·페어링.
- 하루 deep 3~4시간 확보.

**대학원생 (Bimodal)**:
- 평일 오전 4시간 = deep (논문·분석).
- 매주 수요일 = "열린 날" (면담·회의·세미나).
- 논문 마감 2주 전 = Monastic 모드, 모든 외부 약속 취소.

**마케터 (Journalistic)**:
- 회의 많은 직무. 틈새 1시간이 생기면 즉시 고난도 작업 투입.
- 훈련된 사람만 가능. 초보자는 Rhythmic부터.

### 한국 조직·학습 문화와의 마찰

한국 직장·학습 환경에서 deep work 구현 난점과 대응:

| 환경 조건 | 마찰 | 현실적 대응 |
|---|---|---|
| **상시 메신저·카톡 업무** | 즉시 응답 기대 | "오전 2시간 집중, 그 뒤 일괄 응답" 팀 합의. 어렵다면 오전 일찍 또는 점심 직후 조용한 시간만. |
| **회의 과다** | 파편화 | 수요일·금요일 "회의 없는 블록" 개인 캘린더 고정. 협상 불가면 출퇴근 전후 활용. |
| **수험생 독서실·스카이 문화** | 시간 양 자랑 | Deep 시간 총량 추적. 8시간 앉아있어도 deep 2시간일 수 있음. 품질 추적. |
| **인강 몰아듣기·EBS 완주** | 수동 소비 = deep 아님 | 강의는 shallow(정보 도입). Deep은 인출·문제풀이·자기 설명 블록에서. |
| **야자·새벽 기상** | 수면 부족이 인지 손상 | 7시간 수면 > 새벽 기상. 개인 크로노타입에 맞게. |
| **가족·돌봄 부담** | 긴 블록 확보 어려움 | Bimodal(주말 집중) 또는 15~25분 pomodoro 누적. 블록이 짧아도 꾸준이 핵심. |
| **사무실 개방형 좌석** | 환경 통제 약함 | 소음 차단 헤드폰, 회의실 부킹, 재택 활용. 환경 설계가 의지보다 강함. |

## 안티패턴

- **"항상 집중"** — Deep work가 상시 가능한 양식으로 오해. 실제론 하루 ~4시간이 상한(Newport 본인 포함 최상급 지식 노동자도).
- **블록 없이 "집중하자" 의지** — 환경 설계 없이 의지로 극복 시도. 실패 반복.
- **이메일 0 목표** — shallow 제거가 목표가 아님. *비중* 조절.
- **완벽한 monastic 시도** — 대부분 실패. Rhythmic에서 시작.
- **알림만 끄고 폰은 옆에** — 존재 자체가 주의 드래그(Ward et al. 2017).
- **"Slow Productivity"로 책임 회피** — 구조적 과부하인데 개인 집중법만 바꾸려 하면 해결 안 됨.

## 한계

1. **개인 자율이 전제** — 자기 일정 통제권 없는 직무(고객 응대·의료 당직)엔 적용 제한.
2. **근거의 연구 수준** — Newport의 책은 에세이·사례 중심. 통제 실험보단 이론 통합.
3. **문화 차이** — 한국 조직문화(상시 연결·메신저 즉시 응답)에서 개인적 실천엔 마찰.
4. **감정 노동·돌봄 무시** — 관계·가족 맥락에서 "깊은 일 보호"는 이기적일 수 있음. 균형 필요.
5. **번아웃 위험** — 매일 4시간 고강도 deep work는 지속 가능성 의문. 회복·shallow·휴식도 필요.
6. **Cult-like 적용** — "deep 시간 채우기"가 자기 목적화되는 현상.

## 이 프레임워크와 함께 쓰는 것들

- **pomodoro-and-focus** — Deep Work의 *구현 도구*. Pomodoro 단위로 deep block을 구성.
- **deliberate-practice** — deep work의 질적 기준. 단지 집중한다가 아니라 *한계 바로 밖*을 향함.
- **metalearning** — 어떤 일이 나의 deep이고 어떤 일이 shallow인지 자기 분석.
- **zettelkasten / second-brain-para** — Deep work의 산출물 포착·조직.

## 이 프레임워크가 *틀렸을 때*

- 집중 5분도 못 유지 → `pomodoro-and-focus` 먼저 훈련
- 학습 자체가 부족 → `active-recall` / `spaced-repetition`
- 스킬 개선 → `deliberate-practice`
- 정보 조직 문제 → `second-brain-para`
