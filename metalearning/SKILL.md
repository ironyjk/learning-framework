# Metalearning

## 레이어 위치

**L1 (상위 설계)** — 모든 레이어를 고르고 배분하는 메타 레이어. 10% 시간 원칙(Young) — 설계가 학습 자체를 대체하지 않도록. L1 과잉 = 기법 쇼핑·유튜브 학습법 소비.

## 한 줄 요약

**학습 그 자체를 학습하라.** 한 분야 안에서가 아니라, *어떻게 배울지*를 먼저 설계하는 상위 레이어. 인지과학 원리 이해 + 학습 프로젝트 구조화.

## 이론·기원

### Barbara Oakley — *Learning How to Learn* (2014)
- 엔지니어링 교수. 자신이 수학·과학에 약했다가 성인이 되어 공학 전공 전환.
- Coursera 동명 MOOC — 수백만 명 수강한 가장 인기 있는 강좌 중 하나.
- Terrence Sejnowski(신경과학자)와 공저.
- 주요 개념:
  - **Focused / Diffuse mode** — Sejnowski의 신경과학 모티브(default mode network 등). 집중·분산 두 모드 번갈아. [확인 필요: Oakley 원저의 정확한 귀속 출처]
  - **Illusion of competence** — 재독·하이라이트의 함정 (→ `active-recall`).
  - **Chunking** — 전문성의 청크 표상 (→ `chunking`).
  - **Procrastination & pomodoro** — 미루기 신경과학 + 시간 단위 (→ `pomodoro-and-focus`).

### Scott Young — *Ultralearning* (2019)
- 독학자·작가. MIT OpenCourseWare Challenge (1년에 MIT CS 4년치), The Year Without English.
- 학습 프로젝트 9원칙 제시. 사례 중심 실용서.
- 주요 원칙:
  - **Metalearning first** — 주제 학습 전에 *어떻게 배울지* 먼저 맵 그림 (~10% 시간 투자).
  - **Focus** — 집중 블록 확보.
  - **Directness** — 실제 쓸 맥락에서 연습.
  - **Drill** — 약점 집중 반복.
  - **Retrieval** — 인출 연습.
  - **Feedback** — 빠르고 구체적 피드백.
  - **Retention** — 장기 유지 설계.
  - **Intuition** — 깊은 이해·직관.
  - **Experimentation** — 가르쳐 준 대로가 아니라 실험.

두 권은 겹치는 부분과 보완하는 부분이 있음. Oakley는 신경과학·대중 교양, Young은 프로젝트 실행.

## 핵심 개념

### 1. Focused vs Diffuse 모드
- **Focused**: 익숙한 패턴 위 의식적 주의. 연습·문제 풀이·읽기.
- **Diffuse**: 의식 배후의 느슨한 연결. 산책·샤워·자기 직전·지루한 반복 중 떠오르는 통찰.
- 두 모드 *번갈아* 쓰는 것이 창의·어려운 문제 해결의 핵심.
- 막힘 → 잠시 떠남 → 돌아옴 패턴이 과학자·작가 사례에 반복 등장.

**주의**: "좌뇌/우뇌"와 다름. 좌우뇌 학습자 이론은 근거 없음. Focused/diffuse는 신경과학 연구에서 어느 정도 지지.

### 2. Illusion of competence
- 재독·하이라이트·필기 정리 후 "안다" 느낌 ≠ 실제 수행.
- Koriat·Bjork (2005). 자기 평가 ↑ / 실제 회상 → 괴리.
- 깨는 방법: 인출(→ `active-recall`), 설명(→ `feynman-technique`).

### 3. Metalearning Map (Young)
- 새 분야 시작 전 10% 시간을 *지도 그리기*에 투자.
- 물을 질문:
  - **Why**: 왜 배우는가? (instrumental / intrinsic)
  - **What**: 개념(concepts) / 사실(facts) / 절차(procedures) 비중?
  - **How**: 어떤 자료·커리큘럼·사람이 이미 있는가?
- 지도 안 그리고 시작하는 학습은 대개 방황.

### 4. MOC (Map of Content)
- Matuschak·PKM 커뮤니티 용어. 한 주제의 *진입점 노트*.
- 학습 진도·관계 시각화.
- Metalearning 지도의 *살아있는 버전*.

### 5. Directness
- 배울 맥락과 쓸 맥락을 일치시킴.
- 회화를 쓰려면 회화 연습. 문법책만 파면 전이 약함.
- 코드를 쓰려면 코드 작성. 영상 시청만으론 부족.

## 언제 쓰나

- **새 분야 독학 설계** — 코스·튜토리얼을 어떻게 조합할지
- **학습이 계속 실패·정체** — 기법 자체가 문제일 때
- **여러 기법 중 선택 모호** — 메타 레이어가 통합
- **학습 프로젝트 시작 전** — Ultralearning식 프로젝트
- **자기 학습 진단** — 어디가 막히는지

## 실전 프로토콜

### Young식 Metalearning 단계 (1~3주)
1. **동기 명확화** — 왜 배우는가, 얼마나, 언제까지.
2. **선행 조사** — 이 분야 전문가들은 어떻게 배웠나. 블로그·서적·인터뷰·로드맵 수집.
3. **개념/사실/절차 비중 결정**:
   - 예: 프론트엔드 개발 = 개념(많음) + 절차(많음) + 사실(적음) → deliberate practice + feynman 중심
   - 예: 의대 본과 = 사실(많음) + 개념(중간) → SRS + active recall 중심
4. **벤치마크 설정** — 기존 커리큘럼(대학 코스·유명 로드맵) 기준.
5. **수정 = Emphasize / Exclude** — 자기 목적에 맞게 불필요한 부분 빼고 핵심 강조.
6. **일정 설계** — 총 기간·주간 시간·피드백 포인트.

### Oakley식 학습 습관
- 하루 Pomodoro 세션 × N.
- 매 세션 끝에 휴식 = diffuse 모드 시간.
- 어려운 문제 부딪히면 떠나서 diffuse 허용 (잠·산책·샤워).
- 매주 자기 검사(인출 테스트).

### 학습 프로젝트 템플릿
```
프로젝트: [이름]
기간: [N개월]
총 시간: [예상 N시간]
목표(success criterion): [측정 가능하게]
What (내용):
  - 개념: [리스트]
  - 사실: [리스트]
  - 절차: [리스트]
자료:
  - 주교재: [책·코스]
  - 부교재: [블로그·영상]
  - 사람: [튜터·커뮤니티]
기법 조합:
  - [active-recall / SRS / feynman / deliberate-practice 등 배분]
피드백 채널:
  - [시험·코드 리뷰·교사·스터디·AI]
리스크 & 플랜 B:
  - [예상 실패 지점과 대안]
```

### 외부 피드백 채널 설계 (자기주도 ≠ 고립)

자기주도 학습의 가장 큰 실패 원인 = 피드백 채널 부재. 혼자는 blind spot이 굳음. 메타 설계 단계에서 *미리* 확보:

| 채널 | 강점 | 약점 | 한국 맥락 |
|---|---|---|---|
| **교사·코치·튜터** | 즉시 구체 피드백, 정체 돌파 | 비용, 찾기 어려움 | 과외·학원·개인 레슨. 자격증 시험엔 전문 강사 Q&A |
| **스터디·커뮤니티** | 인출 압력, 동료 질문, 유지 동기 | 수준 편차, 관리 비용 | 학원 스터디·오픈채팅·디스코드. 공무원·자격증에 특히 효과 |
| **인강·MOOC** | 체계·반복, 저비용 | 수동 소비 함정(재독형) | 공단기·EBS·인프런·Coursera. *강의 후 인출 의무화* |
| **기출·모의고사** | 시험 형태 일치, 자동 피드백 | 풀이 해설 의존 | 한국 자격증 생태계 성숙 — 기출 5년치가 표준 |
| **AI (Claude·GPT)** | 무한 인내, Feynman 청중, 질문 생성 | 사실 오류 위험, 오용 시 illusion 증폭 | 아래 절 |
| **자기 녹화·리뷰** | 얼버무림 자각 | 외부 시점 부족 | 발표·강의·코딩 세션 |
| **현장 피드백(실행 결과)** | 가장 진실한 신호 | 지연·비용 | 실무 코드·PR·실제 시험·실제 연주 |

**원칙**: 학습 프로젝트 시작 전에 "피드백 1회 받으려면 몇 시간·얼마?"를 계산. 피드백 사이클이 길수록 정체가 길다.

### AI와 함께 학습하는 법 (Feynman 청중화)

AI는 *재독기계*가 아니라 *인출 증폭기*로 써야 한다.

**좋은 프롬프트 패턴** (내가 인출 → AI가 피드백):
- "이 개념을 내가 설명해볼게. 막히거나 틀린 부분을 지적해줘: [내 설명]"
- "[교재] 3장 개념 중 빠진 게 뭘까. 목차 없이 내가 복원해볼게: [내 복원]"
- "이 문제 오답 이유를 내가 분석해볼게: [내 분석]. 놓친 지점을 질문으로 돌려줘."
- "이 주제에 대해 내가 모를 것 같은 반례·엣지케이스 5개 질문해줘." (*pretesting*)
- "이 카드의 빈칸을 내가 답해볼게: [내 답]. 맞는지만 판단하고, 힌트는 단계적으로."

**나쁜 프롬프트 패턴** (AI가 읽어줌 → 내가 재독):
- ✗ "이 챕터 요약해줘" (AI가 인출, 내가 소비)
- ✗ "이 개념 쉽게 설명해줘" (Feynman을 AI가 대신)
- ✗ "이 책 정리해줘" (hover reading)
- ✗ "플래시카드 100장 만들어줘" (카드 만들기 과정 자체가 학습인데 박탈)

**원칙**: *내가 인출하고 AI가 채점한다.* AI가 인출하고 내가 보기만 하면 Dunlosky low utility로 돌아간다. AI에게 "내가 먼저 답할게, 너는 판정만 해"를 명시적으로 지시.

**한계**:
- AI는 사실 오류·hallucination 가능. 고위험 영역(의학·법률·자격증 법령)은 *반드시* 공식 자료로 교차 검증.
- AI와의 대화는 spacing 없이 한 세션에 끝나면 유지 약함. SRS와 결합 필요.
- AI가 "잘했어요"를 남발하면 위험. 엄격 판정 모드를 명시("관대하지 말고 구체적 구멍 지적").

## 한국 맥락 예시

**프론트엔드 → 백엔드 전환 (6개월)**:
- Why: 풀스택으로 역량 확장, 시장 가치 ↑
- What: 개념(HTTP·DB·시스템 설계 많음) + 절차(언어·프레임워크 많음) + 사실(API 문법 중간)
- How: 『Designing Data-Intensive Applications』 + 실무 프로젝트 + System Design 인터뷰 + 멘토 코드 리뷰
- Mix:
  - 개념 이해 → feynman + evergreen-notes
  - 문법·라이브러리 → active recall + 실제 코드 작성(directness)
  - 시스템 설계 → deliberate practice (주 2회 설계 + 피드백)
  - 집중 블록 → deep-work rhythmic

**IFAT 박람회 준비 (1개월)**:
- Why: 물처리 장비 시장 조사
- What: 사실(회사·제품 정보) + 개념(수처리 기술 원리) + 절차(박람회 질문)
- Mix:
  - 독일어 기본 어휘 → SRS
  - 수처리 기술 원리 → feynman + 논문 정리(zettelkasten light)
  - 미팅 시나리오 → deliberate practice (모의 미팅 + 피드백)

## 안티패턴

- **기법 쇼핑** — 매주 새 기법 시도, 정착 없음. Metalearning은 도구지 수집이 아님.
- **지도 그리기에 영원** — 10% 원칙 무시, 준비만 하다 시작 못 함.
- **유튜브 학습법만** — 학습법 영상 100개 시청이 학습 아님. Illusion of competence 메타버전.
- **한 책으로 해결 기대** — *Learning How to Learn*·*Ultralearning* 읽기만으로 바뀌지 않음. 실행 없이 지식만.
- **도구 세팅 피로** — Obsidian 플러그인 세팅 3일, Anki 덱 다운로드 후 방치, Notion 대시보드 구성에 주말 소진. 도구 세팅이 학습 회피 구실이 되는 패턴.
- **잘못된 "법칙" 내재화**:
  - "10,000시간 법칙" — Ericsson 본인이 Gladwell 반박 (→ `deliberate-practice`)
  - "학습 유형"(VAK·시각형/청각형/운동형) — Pashler et al. (2008) *Psychological Science in the Public Interest* 비판 리뷰에서 *"meshing hypothesis"를 지지하는 실증 근거가 없다*고 결론
  - "우뇌형/좌뇌형" — Nielsen et al. (2013) fMRI 1,011명 연구에서 좌우뇌 선호 없음
  - "속독" — Rayner et al. (2016) *Psychological Science in the Public Interest* — 이해도 유지하며 빠르게 읽는 방법의 실증 없음. 속도↑는 이해도↓와 교환 관계.
  - "1분/60초 기억법"·"천재의 비밀" — 대중서·유튜브 과장
  - **한국형 오개념**:
    - "수능은 암기력" — 실제론 유형 식별·절차 + 시간 압박 하 인출. 인터리빙·retrieval이 핵심.
    - "문제집 10회독" — Dunlosky low utility. 회독은 익숙함만 증가.
    - "새벽 4시 기상이 성공의 비결" — 근거 없음. 개인 크로노타입 차이 크고 수면 부족이 인지 손상.
    - "Anki는 의대생만 쓴다" — 도메인 무관. 사실 암기가 필요한 모든 학습에 유효.
    - "Zettelkasten 안 하면 지식인 아니다" — 도구·시스템 물신화. Luhmann 개인 사례가 모두에게 전이되지 않음.
    - "인강만 완주하면 된다" — 수동 소비. 인출 없으면 유지 안 됨.
- **재능 이분법** — "나는 수학머리 없다" 고정 마인드셋. Oakley 본인이 반례(40대에 공학 전공). → `growth-mindset` 참조.

## 한계

1. **메타 지식이 실제 학습 대체 못 함** — 알아도 안 하면 소용없음. 결국 시간·반복.
2. **분야별 차이** — 메타 원리는 공통이지만 세부 기법은 분야별. 의대·음악·언어·프로그래밍 각기 다름.
3. **자기 기만 경계** — 메타 이론을 읽으며 "나는 이해했다"는 착각. *실제 행동*이 지표.
4. **Young의 일화 증거** — *Ultralearning* 사례들은 드라마틱하지만 일반화 주의. 배경·시간·자원 차이.
5. **Oakley의 신경과학 대중화** — 일부 개념(좌뇌/우뇌 같은 오개념 언급)은 신중히. 본인이 이후 더 정확히 수정.
6. **메타 학습에만 몰입** — 학습법 공부가 학습 회피가 될 수 있음.

## 이 프레임워크와 함께 쓰는 것들

- **모든 다른 프레임워크** — Metalearning은 *상위 레이어*. 조합을 선택하는 역할.
- **deliberate-practice** — Young의 Directness·Drill·Feedback이 Ericsson과 맞물림.
- **active-recall + spaced-repetition** — 사실·개념 유지의 기본 엔진.
- **feynman-technique** — 개념 이해·illusion 깨기.
- **zettelkasten / evergreen-notes** — 학습 결과물 축적.
- **deep-work / pomodoro-and-focus** — 시간·환경 레이어.

## 이 프레임워크가 *틀렸을 때*

- 기법이 아닌 집중·수면·건강 문제 → 기본 생활부터
- 구체적 스킬 연습 중 → `deliberate-practice` 직접
- 시험이 코앞(2주 이하) → SRS + active recall 직행, 메타 설계 미루기
- 이해 구멍 → `feynman-technique` 즉시
